# Question Capital Map
Version: v1
Vertical: K-Beauty Skincare

---

## 1. 문서 목적

Question Capital은 버티컬이 점유해야 할 질문 영토다.
이 문서는 공통 질문, 니치 질문, 브랜드 전용 질문을 계층화한다.

---

## 2. Layer 구조

### Layer A. Common Question Capital
버티컬 전체 공통 질문

### Layer B. Niche Question Capital
특정 고민/상황/세그먼트 특화 질문

### Layer C. Brand-specific Question Capital
특정 브랜드/제품/라인/증거와 연결되는 질문

## 4. 주요 노드

### QC-01 Problem / Concern
- 건조/당김
- 민감/자극
- 장벽 약화
- 트러블/피지
- 모공
- 칙칙함/톤
- 탄력
- 홍조
- 각질/결

### QC-02 Fit / Eligibility
- 피부 타입 적합성
- 민감 피부 적합성
- 초보자 적합성
- 계절 적합성
- 메이크업 병행 적합성
- 고기능 성분 적합성

### QC-03 Product Type / Routine Role
- 클렌저 역할
- 토너/패드 역할
- 에센스/세럼/앰플 차이
- 로션/크림/젤 차이
- 최소 루틴 구성

### QC-04 Ingredient / Mechanism
- 보습/장벽 성분
- 진정 성분
- 미백/톤 성분
- 각질 관리 성분
- 트러블/피지 성분
- 성분 조합 궁합

### QC-05 Compare / Choice
- 제품군 간 비교
- 성분 간 비교
- 제품 vs 제품
- 브랜드 vs 브랜드
- 가격/텍스처/목적별 비교

### QC-06 Usage / Routine Execution
- 순서
- 빈도
- 아침/저녁
- 병행 사용
- 도입 스케줄
- 루틴 리셋

### QC-07 Proof / Trust
- claim
- evidence
- reviewer
- test/data
- updatedAt
- changelog

### QC-08 Boundary / Safety
- 비적합 대상
- 주의 성분 조합
- 기대치 경계
- 자극 대응
- 상담 필요 상황

### QC-09 Brand / Product Specific
- 브랜드 핵심 정답
- 제품 적합성
- 제품 비교
- 제품 근거
- 브랜드 루틴
- 브랜드 action

### QC-10 Action / Commerce
- 1~3개 제품 시작
- 루틴 저장
- 샘플/체험
- 구매 vs 상담
- 예산별 추천
- 재구매/업셀

## 6. 운영 원칙

- 질문은 페이지 제목이 아니라 실제 질문 형태를 유지
- 공통 / 니치 / 브랜드 전용을 분리
- 모든 질문은 proof / boundary / action으로 닫힐 수 있어야 함
- no dead-end를 만족해야 함
