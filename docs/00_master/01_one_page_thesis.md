## 1. 한 줄 정의

우리는 브랜드와 버티컬이 AI 검색 시대에 공식 정답을 운영할 수 있도록,
Question-first / Truth-first 기반의 Answer Operating System을 만들고,
시장에는 Vertical AEO Hub 제품으로 먼저 판매한다.

---

## 2. 문제 정의

기존 홈페이지와 콘텐츠 운영은 아래 문제를 동시에 가진다.

1. 질문 구조가 약하다
2. 정답 정본과 마케팅 문구가 섞여 있다
3. 비교 / 근거 / 경계 / 행동이 분리되어 있지 않다
4. AI 검색과 인간 탐색 모두에 최적화되어 있지 않다
5. 운영 품질을 측정하고 고치는 체계가 없다
6. 브랜드마다 유사 구조를 반복 제작한다

---

## 4. 제품화 전략

- 엔진은 범용으로 만든다
- 제품화는 버티컬 단위로 한다
- 판매 단위는 골디락스한 브랜드 도메인이다
- 허브 안에 브랜드 Mini B-SSoT를 포함한다
- 운영 구독(D-MRI/B-MRI)을 recurring revenue로 둔다
- 장기적으로 Factory / Portfolio 구조로 확장한다

---

## 5. 왜 강한가

이 구조는 다음을 동시에 갖는다.

- SaaS 코어
- 버티컬 데이터 해자
- 브랜드별 반복 생산 가능성
- 운영 구독 수익
- answer commerce 연결성

즉, 단발 구축이 아니라
Answer OS + Hub + Observatory + Factory 모델이다.
